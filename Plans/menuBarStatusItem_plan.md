# WorkPulse 메뉴 막대 상태 아이템 표시 계획 (`menuBarStatusItem_plan.md`)

## 요약
- 기준 대상은 현재 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 앱은 일반 창 앱에 더해 macOS 상단 메뉴 막대(Status Bar)에 표시되어야 한다.
- 상단 노출은 `NSStatusBar` / `NSStatusItem` 기반으로 구현한다.
- 상태 아이템 상호작용 범위는 우선 "클릭 시 메인 창 표시 또는 전면 복귀"로 한정한다.
- 이번 범위에서는 측정 기능, 화면 레이아웃, 데이터 모델 로직은 변경하지 않는다.

### 목표 (Goal)
WorkPulse를 실행하면 사용자가 macOS 상단 메뉴 막대에서 앱을 즉시 인지할 수 있고, 상태 아이템 클릭만으로 메인 창에 접근할 수 있도록 한다.

### 제약 사항 (Constraints)
- 핵심 표시/상태 아이템 로직은 `AppDelegate` 중심의 최소 변경으로 구현한다.
- 기존 macOS 앱 실행 및 메인 창 표시 동작을 깨지 않는다.
- 메뉴 막대 표시를 위해 불필요한 아키텍처 확장이나 별도 상태 관리 계층을 추가하지 않는다.
- 첫 단계에서는 드롭다운 메뉴, 팝오버, 설정 화면 같은 부가 UX는 범위에서 제외한다.
- Dock 아이콘 제거(`LSUIElement`, accessory policy)는 이번 요구사항에 명시되지 않았으므로 기본 범위에서 제외한다.

## 공개 인터페이스/타입 변경(계획 기준)
1. `AppDelegate`에 메뉴 막대 상태 아이템 보관 속성을 추가한다.
- `statusItem: NSStatusItem?`

2. `AppDelegate`에 상태 아이템 초기화 메서드를 추가한다.
- 예: `configureStatusItem()`

3. `AppDelegate`에 상태 아이템 클릭 처리 메서드를 추가한다.
- 예: `handleStatusItemClick()`

4. 기존 `ViewController` 공개 인터페이스는 유지한다.
- 창을 다시 표시하는 책임은 `AppDelegate`에 둔다.

## 행동 변경 (Behavioral Changes - TDD)
*지시가 있을 때마다 아래의 체크되지 않은 첫 번째 테스트를 찾아 Red 테스트를 작성하고, 이를 통과시키는 최소 코드만 작성한다.*

- [x] 앱 실행 시 메뉴 막대에 `NSStatusItem` 1개가 생성된다.
- [x] 상태 아이템 버튼은 비어 있지 않은 이미지 또는 제목 중 최소 1개를 가져 사용자가 식별 가능하다.
- [x] 상태 아이템 클릭 시 메인 창이 없으면 생성되고, 이미 있으면 전면으로 복귀한다.
- [x] 메인 창이 뒤로 가 있거나 다른 앱 뒤에 있을 때 상태 아이템 클릭 시 앱이 활성화된다.
- [x] 마지막 창을 닫아도 앱이 즉시 종료되지 않고 메뉴 막대 항목은 유지된다.
- [x] 앱을 다시 활성화했을 때 동일한 상태 아이템 인스턴스가 중복 생성되지 않는다.
- [x] 상태 아이템 추가가 기존 macOS 앱 빌드 성공을 깨지 않는다.

## 구조 변경 (Structural Changes - Tidy First)
*행동 테스트 Green 이후 리팩터링한다.*

- [x] 창 생성/복원 로직을 `AppDelegate` 내부 단일 메서드로 정리한다.
- [x] 상태 아이템 설정 로직을 `applicationDidFinishLaunching` 본문에서 분리한다.
- [x] 상태 아이템 액션에서 직접 창 생성 세부사항을 다루지 않도록 공통 메서드로 위임한다.
- [x] 메뉴 막대 표시 요구사항과 무관한 View 계층 수정은 하지 않는다.
- [x] Dock 정책 변경이 필요해질 경우 현재 계획과 분리된 후속 계획으로 다룬다.

## 테스트 케이스 및 시나리오
1. 기본 실행
- Given: 앱이 처음 실행됨
- When: `applicationDidFinishLaunching` 완료
- Then: 메뉴 막대에 상태 아이템 1개가 존재

2. 클릭으로 창 복귀
- Given: 메인 창이 이미 생성되어 있으나 비활성 상태
- When: 상태 아이템 클릭
- Then: 앱이 활성화되고 메인 창이 key window 로 올라옴

3. 창 닫기 후 유지
- Given: 메인 창을 사용자가 닫음
- When: 앱이 유휴 상태가 됨
- Then: 앱은 종료되지 않고 메뉴 막대 항목은 유지

4. 재실행/재활성화 중복 방지
- Given: 상태 아이템이 이미 설정된 실행 상태
- When: 창을 다시 열거나 앱을 재활성화
- Then: 상태 아이템이 중복으로 추가되지 않음

5. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “상단에 프로그램이 떠있어야 한다”는 요구는 우선 메뉴 막대 상태 아이템 표시로 해석한다.
2. 첫 단계에서는 드롭다운 메뉴보다 "아이콘 노출 + 클릭 시 창 표시"가 우선이다.
3. Dock 아이콘 유지 여부는 현재 요구사항만으로는 변경하지 않는다.
4. 메뉴 막대 아이콘 스타일은 macOS 표준 template image를 기본값으로 사용한다.
5. 이후 사용자가 원하면 후속 단계에서 `LSUIElement` 기반 순수 메뉴 바 앱으로 확장할 수 있다.
