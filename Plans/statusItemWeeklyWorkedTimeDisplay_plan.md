# WorkPulse 상태 아이템 이번 주 근무시간 표시 계획 (`statusItemWeeklyWorkedTimeDisplay_plan.md`)

## 요약
- 기준 대상은 `menuBarStatusItem_plan.md`, `statusItemAttendanceTimeInput_plan.md`, `statusItemWorkedTimeDisplay_plan.md`를 전제로 한 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 사용자는 메뉴 막대 상태 아이템 또는 상태 아이템 클릭 후 열리는 팝오버에서 이번 주 누적 근무시간을 확인할 수 있어야 한다.
- 이번 주 근무시간은 로컬에 저장된 일별 출근/퇴근 기록을 주 단위로 합산해 계산한다.
- 표시 범위는 현재 주(`Calendar` 기준 week interval)로 한정한다.
- 이번 범위에서는 주간 목표시간 비교, 초과근무 분리, 월간 통계는 제외한다.

### 목표 (Goal)
사용자가 WorkPulse를 열었을 때 이번 주에 누적된 총 근무시간을 즉시 확인할 수 있도록 하고, 근무 기록 저장 시 주간 합계가 바로 갱신되도록 한다.

### 제약 사항 (Constraints)
- 주간 합계 계산 로직은 Foundation 기반 순수 로직으로 테스트 가능해야 한다.
- AppKit 의존은 상태 아이템/팝오버 표시 계층에 한정한다.
- 주간 합계는 로컬에 저장된 일별 출근/퇴근 완료 기록만 집계 대상으로 삼는다.
- 진행 중 근무(출근시간만 있고 퇴근시간이 없는 상태)를 주간 합계에 포함할지 여부는 명시적으로 정의해야 한다.
- 첫 단계에서는 단일 사용자, 단일 로컬 저장소만 다룬다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 일별 근무 기록 모델을 추가한다.
- 예: `AttendanceRecord`
- 속성 예시: `date`, `startTime`, `endTime`

2. 기록 목록 저장 책임을 분리하는 저장소 타입을 추가한다.
- 예: `AttendanceRecordStore`
- 책임: 저장, 조회, 특정 날짜 갱신

3. 주간 합계 계산용 순수 타입을 추가한다.
- 예: `WeeklyWorkedTimeCalculator`

4. 상태 아이템/팝오버에서 재사용할 표시 모델을 추가한다.
- 예: `WeeklyWorkedTimeDisplayModel`
- 속성 예시: `text`, `isPlaceholder`

- [x] 이번 주에 저장된 근무 기록이 없으면 이번 주 근무시간은 placeholder로 표시된다.
- [x] 이번 주에 하루 근무 기록 1개가 있으면 해당 시간만큼 주간 합계가 표시된다.
- [x] 이번 주에 여러 날의 근무 기록이 있으면 각 일자의 근무시간 합계가 표시된다.
- [x] 이번 주 범위를 벗어난 지난주/다음주 기록은 주간 합계에서 제외된다.
- [x] 동일 날짜의 기록이 수정되면 이전 값이 중복 합산되지 않고 최신 값 기준으로 다시 계산된다.
- [x] 진행 중 근무를 포함하도록 정의한 경우 현재 시각 기준 값이 반영되고, 제외하도록 정의한 경우 확정 기록만 합산된다.
- [x] 출근/퇴근시간 저장 또는 수정 후 이번 주 근무시간 표시가 즉시 갱신된다.
- [x] 주간 합계 표시 추가가 기존 macOS 앱 빌드 성공을 깨지 않는다.

## 테스트 케이스 및 시나리오
1. 기록 없음
- Given: 이번 주에 저장된 근무 기록이 없음
- When: 주간 합계 계산
- Then: placeholder 문자열 표시

2. 단일 일자 집계
- Given: 이번 주 월요일 근무 기록 1개가 저장됨
- When: 주간 합계 계산
- Then: 해당 일자의 근무시간이 그대로 표시

3. 복수 일자 합계
- Given: 이번 주에 여러 날짜의 근무 기록이 저장됨
- When: 주간 합계 계산
- Then: 각 날짜 근무시간의 합이 표시

4. 주간 경계 제외
- Given: 지난주 기록과 이번 주 기록이 함께 저장됨
- When: 이번 주 합계 계산
- Then: 이번 주 기록만 집계

5. 기록 수정 반영
- Given: 오늘 기록이 이미 저장되어 있음
- When: 사용자가 오늘 출근/퇴근시간을 수정
- Then: 주간 합계가 최신 기록 기준으로 즉시 갱신

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “이번주”는 사용자의 현재 로케일/캘린더 기준 `weekOfYear` 구간으로 해석한다.
2. 주간 합계는 기본적으로 일별 `퇴근시간 - 출근시간`의 합으로 계산한다.
3. 같은 날짜 기록은 1개만 유지하고, 수정 시 덮어쓴다.
4. 진행 중 근무 포함 여부는 구현 전 명시해야 하며, 별도 지시가 없으면 확정된 출근/퇴근 기록만 집계하는 보수적 기본값을 사용한다.
5. 휴게시간 차감, 자정 넘김, 다중 세션 분할은 이번 범위에서 다루지 않는다.
