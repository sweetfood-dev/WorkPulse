# WorkPulse 상태 아이템 출근/퇴근시간 입력 계획 (`statusItemAttendanceTimeInput_plan.md`)

## 요약
- 기준 대상은 macOS 전용 `WorkPulse` AppKit 앱과 메뉴 막대 상태 아이템(Status Item) 요구사항으로 고정한다.
- 사용자는 상단 메뉴 막대 아이콘을 클릭하여 출근시간과 퇴근시간을 입력할 수 있어야 한다.
- 첫 단계의 입력 UI는 상태 아이템 클릭 시 열리는 경량 팝오버(`NSPopover`)로 한정한다.
- 입력 컨트롤은 macOS 기본 시간 선택 컴포넌트를 사용한다.
- 이번 범위에서는 서버 연동이나 외부 데이터 동기화는 제외하고 로컬 저장만 다룬다.

### 목표 (Goal)
사용자가 WorkPulse 메뉴 막대 아이콘을 클릭했을 때 출근시간과 퇴근시간을 직관적으로 입력하고, 다시 열어도 마지막 입력값을 확인할 수 있도록 한다.

### 제약 사항 (Constraints)
- 입력 진입점은 메뉴 막대 상태 아이템 클릭 하나로 고정한다.
- 시간 입력 UX는 AppKit 기본 컴포넌트로 구성하고 커스텀 캘린더/복합 위젯은 추가하지 않는다.
- 저장은 로컬 범위로 제한하고 네트워크, 계정, 백엔드 API는 다루지 않는다.
- 기존 메인 창 렌더링 경로와 측정 관련 화면 로직은 변경하지 않는다.
- 첫 단계에서는 “출근시간/퇴근시간 입력 및 재표시”까지만 목표로 하고, 근무 시간 계산/통계/자동 기록은 제외한다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 출근/퇴근 입력 상태를 표현하는 순수 모델을 추가한다.
- 예: `AttendanceTimeInputModel`
- 속성 예시: `startTime`, `endTime`

2. 로컬 저장 책임을 분리하는 저장 프로토콜/구현을 추가한다.
- 예: `AttendanceTimeStore`
- 기본 구현은 `UserDefaults` 기반

3. 메뉴 막대 팝오버 전용 ViewController를 추가한다.
- 예: `AttendanceTimePopoverViewController`
- 책임: 입력값 표시, 수정, 저장 액션 전달

4. 상태 아이템을 관리하는 진입점은 `AppDelegate`에 둔다.
- 클릭 시 팝오버 표시/숨김 제어

## 행동 변경 (Behavioral Changes - TDD)
*지시가 있을 때마다 아래의 체크되지 않은 첫 번째 테스트를 찾아 Red 테스트를 작성하고, 이를 통과시키는 최소 코드만 작성한다.*

- [x] 메뉴 막대 상태 아이템 클릭 시 출근시간/퇴근시간 입력 팝오버가 열린다.
- [x] 팝오버 최초 진입 시 저장된 값이 없으면 출근시간/퇴근시간 입력 필드는 비어 있거나 기본값 상태로 표시된다.
- [x] 사용자가 출근시간을 입력하고 저장하면 로컬 저장소에 반영된다.
- [x] 사용자가 퇴근시간을 입력하고 저장하면 로컬 저장소에 반영된다.
- [x] 출근시간과 퇴근시간을 모두 저장한 뒤 팝오버를 다시 열면 마지막 저장값이 다시 표시된다.
- [x] 팝오버를 닫았다가 다시 열어도 저장되지 않은 임시 입력은 확정값을 덮어쓰지 않는다.
- [x] 저장 로직 추가가 기존 macOS 앱 빌드를 깨지 않는다.

## 구조 변경 (Structural Changes - Tidy First)
*행동 테스트 Green 이후 리팩터링한다.*

- [x] 시간 입력/표시 모델을 ViewController 바깥의 순수 타입으로 분리한다.
- [x] `UserDefaults` 직접 접근을 저장 구현으로 캡슐화한다.
- [x] 상태 아이템 클릭 처리와 팝오버 생성 로직을 `AppDelegate` 내부 공통 메서드로 정리한다.
- [x] 팝오버 UI 구성과 저장 액션 연결을 분리해 ViewController 책임을 축소한다.
- [x] 시간 포맷팅이 필요하면 공통 포맷터 접근 경로를 단일화한다. (현재 범위에서 시간 포맷팅 미사용)

## 테스트 케이스 및 시나리오
1. 팝오버 열기
- Given: 메뉴 막대 상태 아이템이 존재
- When: 사용자가 상태 아이템을 클릭
- Then: 출근/퇴근시간 입력 팝오버가 표시

2. 출근시간 저장
- Given: 팝오버가 열려 있음
- When: 사용자가 출근시간을 선택하고 저장
- Then: 저장소에 출근시간이 기록

3. 퇴근시간 저장
- Given: 팝오버가 열려 있음
- When: 사용자가 퇴근시간을 선택하고 저장
- Then: 저장소에 퇴근시간이 기록

4. 재오픈 시 값 복원
- Given: 출근시간과 퇴근시간이 이미 저장됨
- When: 팝오버를 닫았다가 다시 엶
- Then: 저장된 값이 입력 필드에 다시 표시

5. 저장하지 않은 변경 무시
- Given: 저장된 값이 존재
- When: 사용자가 시간을 바꾸고 저장 없이 팝오버를 닫음
- Then: 기존 저장값은 유지

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “아이콘을 클릭”은 메뉴 막대 상태 아이템 클릭으로 해석한다.
2. 입력 UI는 첫 단계에서 별도 창이 아니라 팝오버로 제공한다.
3. 시간은 로컬 단말 기준으로 다루며, 타임존 변환은 이번 범위에서 제외한다.
4. 저장은 로컬(`UserDefaults`)만 다루고 서버 동기화는 하지 않는다.
5. 야간 근무/자정 넘김 검증 규칙은 요구사항에 없으므로 이번 범위에서 강제하지 않는다.
