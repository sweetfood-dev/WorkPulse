# WorkPulse 상태 아이템 현재 근무시간 표시 계획 (`statusItemWorkedTimeDisplay_plan.md`)

## 요약
- 기준 대상은 `menuBarStatusItem_plan.md`와 `statusItemAttendanceTimeInput_plan.md`를 전제로 한 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 사용자는 메뉴 막대 상태 아이템을 통해 현재까지 일한 시간을 즉시 확인할 수 있어야 한다.
- 현재 근무시간 표시는 메뉴 막대 상태 아이템 본문과 팝오버 상단 요약 영역에서 동일한 기준값을 사용한다.
- 계산 기준 데이터는 로컬에 저장된 출근시간과 퇴근시간이다.
- 이번 범위에서는 휴게시간 차감, 연장근무 규칙, 다중 근무 세션 합산은 제외한다.

### 목표 (Goal)
사용자가 WorkPulse를 실행한 동안 메뉴 막대에서 현재 누적 근무시간을 한눈에 확인하고, 상태 아이템을 클릭해 열리는 입력 UI에서도 동일한 시간을 확인할 수 있도록 한다.

### 제약 사항 (Constraints)
- 핵심 근무시간 계산 로직은 Foundation 기반 순수 로직으로 테스트 가능해야 한다.
- AppKit 의존은 상태 아이템/팝오버 표시 계층에 한정한다.
- 출근/퇴근시간 저장 방식은 기존 계획의 로컬 저장소(`UserDefaults` 기반)를 그대로 사용한다.
- 첫 단계에서는 단일 출근시간과 단일 퇴근시간만 기준으로 계산한다.
- 계산 결과 포맷은 메뉴 막대 폭을 고려한 짧은 문자열 형식으로 제한한다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 현재 근무시간 계산용 순수 타입을 추가한다.
- 예: `WorkedTimeCalculator`

2. 표시용 Render Model을 추가한다.
- 예: `WorkedTimeDisplayModel`
- 속성 예시: `text`, `isPlaceholder`

3. 현재 시각 주입용 추상화를 추가한다.
- 예: `NowProviding`
- 실구현은 시스템 현재 시각 사용

4. 상태 아이템과 팝오버가 같은 표시값을 사용하도록 공통 업데이트 경로를 추가한다.
- 예: `refreshWorkedTimeDisplay()`

## 행동 변경 (Behavioral Changes - TDD)
*지시가 있을 때마다 아래의 체크되지 않은 첫 번째 테스트를 찾아 Red 테스트를 작성하고, 이를 통과시키는 최소 코드만 작성한다.*

- [x] 출근시간과 퇴근시간이 모두 없으면 현재 근무시간은 placeholder로 표시된다.
- [x] 출근시간만 있고 퇴근시간이 없으면 현재 시각 기준 경과 시간이 표시된다.
- [x] 출근시간과 퇴근시간이 모두 있으면 `퇴근시간 - 출근시간` 결과가 표시된다.
- [x] 출근시간만 있는 진행 중 상태에서는 현재 시각이 변하면 표시값도 갱신된다.
- [x] 출근/퇴근시간을 저장하거나 수정하면 현재 근무시간 표시가 즉시 재계산된다.
- [x] 상태 아이템 텍스트와 팝오버 요약 텍스트는 항상 동일한 계산 결과를 사용한다.
- [x] 근무시간 표시 추가가 기존 macOS 앱 빌드 성공을 깨지 않는다.

## 구조 변경 (Structural Changes - Tidy First)
*행동 테스트 Green 이후 리팩터링한다.*

- [x] 근무시간 계산을 View/AppDelegate 바깥의 순수 계산 타입으로 분리한다.
- [x] 현재 시각 접근을 직접 `Date()` 호출하지 않도록 추상화한다.
- [x] 상태 아이템 텍스트 갱신과 팝오버 요약 갱신을 단일 경로로 합친다.
- [x] 표시 포맷터를 공통 유틸리티 또는 전용 타입으로 분리한다.
- [x] 시간 입력 저장 완료 후 표시 갱신이 누락되지 않도록 저장 후 공통 refresh 호출로 정리한다.

## 테스트 케이스 및 시나리오
1. 출근 전 상태
- Given: 출근시간과 퇴근시간이 없음
- When: 현재 근무시간 표시 계산
- Then: placeholder 문자열 표시

2. 근무 진행 중
- Given: 출근시간만 저장됨
- When: 현재 시각 기준 표시 계산
- Then: `현재 시각 - 출근시간` 결과 표시

3. 근무 종료 후
- Given: 출근시간과 퇴근시간이 모두 저장됨
- When: 현재 근무시간 표시 계산
- Then: `퇴근시간 - 출근시간` 고정 결과 표시

4. 시간 변경 반영
- Given: 출근시간만 저장된 상태
- When: 시간이 1분 이상 경과
- Then: 표시 문자열이 새로운 경과 시간으로 갱신

5. 동일 값 재사용
- Given: 상태 아이템과 팝오버가 모두 존재
- When: 현재 근무시간 표시가 갱신
- Then: 두 UI가 동일 문자열을 표시

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “현재 일한 시간”은 기본적으로 `출근시간 ~ 현재시각` 또는 `출근시간 ~ 퇴근시간` 차이로 해석한다.
2. 근무시간 표시는 메뉴 막대 폭을 고려해 기본 포맷을 `HH:mm` 형태의 compact 문자열로 사용한다.
3. 출근시간이 없으면 현재 근무시간은 계산할 수 없으므로 placeholder를 표시한다.
4. 퇴근시간이 저장되면 이후에는 실시간 증가가 아니라 확정 근무시간을 표시한다.
5. 휴게시간 차감, 자정 넘김, 다중 세션 합산은 이번 범위에서 다루지 않는다.
