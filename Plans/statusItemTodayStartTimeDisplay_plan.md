# WorkPulse 상태 아이템 오늘 출근시간 표시 계획 (`statusItemTodayStartTimeDisplay_plan.md`)

## 요약
- 기준 대상은 `menuBarStatusItem_plan.md`, `statusItemAttendanceTimeInput_plan.md`, `statusItemWorkedTimeDisplay_plan.md`를 전제로 한 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 오늘 출근시간이 입력되어 있다면 사용자는 이를 즉시 확인할 수 있어야 한다.
- 첫 단계의 표시 위치는 상태 아이템 클릭 후 열리는 팝오버의 요약 영역으로 한정한다.
- 표시 데이터는 로컬에 저장된 오늘 날짜의 출근시간을 사용한다.
- 이번 범위에서는 출근시간 수정 이력, 다중 출근 세션, 서버 동기화는 제외한다.

### 목표 (Goal)
사용자가 오늘 출근시간을 입력한 뒤 메뉴 막대 아이콘을 클릭하면, 팝오버에서 오늘 출근한 시각을 바로 확인할 수 있도록 한다.

### 제약 사항 (Constraints)
- 오늘 출근시간 판별 로직은 Foundation 기반 순수 로직으로 테스트 가능해야 한다.
- AppKit 의존은 팝오버 표시 계층에 한정한다.
- 오늘 날짜 판단은 사용자의 현재 로컬 캘린더/타임존 기준으로 처리한다.
- 첫 단계에서는 오늘 출근시간이 있을 때만 표시하고, 없으면 placeholder 또는 숨김 상태로 처리한다.
- 메뉴 막대 본문에는 공간 제약이 있으므로 기본 범위에서는 오늘 출근시간을 직접 넣지 않는다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 오늘 출근시간 표시용 Render Model을 추가한다.
- 예: `TodayStartTimeDisplayModel`
- 속성 예시: `text`, `isHidden`, `isPlaceholder`

2. 저장된 근무 기록에서 오늘 기록을 조회하는 순수 로직 또는 저장소 메서드를 추가한다.
- 예: `todayRecord()`

3. 팝오버 요약 UI에 오늘 출근시간 표시 영역을 추가한다.
- 예: "오늘 출근: 09:03"

4. 출근시간 저장 후 오늘 출근시간 표시를 갱신하는 공통 경로를 추가한다.
- 예: `refreshTodayStartTimeDisplay()`

## 행동 변경 (Behavioral Changes - TDD)
*지시가 있을 때마다 아래의 체크되지 않은 첫 번째 테스트를 찾아 Red 테스트를 작성하고, 이를 통과시키는 최소 코드만 작성한다.*

- [ ] 오늘 날짜의 출근시간이 저장되어 있지 않으면 오늘 출근시간 표시는 placeholder 또는 숨김 상태가 된다.
- [ ] 오늘 날짜의 출근시간이 저장되어 있으면 해당 시각이 표시된다.
- [ ] 어제 또는 다른 날짜의 출근시간만 존재하면 오늘 출근시간 표시에는 반영되지 않는다.
- [ ] 오늘 출근시간을 새로 저장하면 표시가 즉시 갱신된다.
- [ ] 오늘 출근시간을 수정하면 이전 값이 남지 않고 최신 값으로 갱신된다.
- [ ] 오늘 출근시간 표시 추가가 현재 근무시간/이번 주 근무시간 표시를 깨지 않는다.
- [ ] 오늘 출근시간 표시 추가가 기존 macOS 앱 빌드 성공을 깨지 않는다.

## 구조 변경 (Structural Changes - Tidy First)
*행동 테스트 Green 이후 리팩터링한다.*

- [ ] 오늘 날짜 판별 로직을 View/AppDelegate 바깥의 순수 경로로 분리한다.
- [ ] 저장소에서 오늘 기록 조회를 공통 메서드로 정리한다.
- [ ] 팝오버 요약 값 갱신을 단일 refresh 경로로 통합한다.
- [ ] 시간 포맷터가 기존 근무시간 표시와 공유 가능하면 공통 포맷터로 정리한다.
- [ ] 오늘 출근시간 표시와 다른 요약 표시(current/weekly)의 결합도를 낮춘다.

## 테스트 케이스 및 시나리오
1. 오늘 기록 없음
- Given: 오늘 날짜의 출근시간이 저장되지 않음
- When: 오늘 출근시간 표시 계산
- Then: placeholder 또는 숨김 상태

2. 오늘 기록 존재
- Given: 오늘 출근시간이 저장됨
- When: 팝오버 표시값 계산
- Then: 저장된 오늘 출근시각이 표시

3. 다른 날짜 기록 제외
- Given: 어제 출근시간만 저장됨
- When: 오늘 출근시간 표시 계산
- Then: 오늘 출근시간은 표시되지 않음

4. 저장 후 즉시 반영
- Given: 팝오버가 열려 있음
- When: 사용자가 오늘 출근시간을 저장
- Then: 요약 영역의 오늘 출근시간이 즉시 갱신

5. 수정 반영
- Given: 오늘 출근시간이 이미 저장됨
- When: 사용자가 오늘 출근시간을 다른 시각으로 수정 후 저장
- Then: 표시값이 최신 시각으로 교체

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “오늘 출근한 시간”은 오늘 날짜의 출근 시각(timestamp)으로 해석한다.
2. 첫 단계의 표시 위치는 메뉴 막대 아이콘 클릭 후 열리는 팝오버 요약 영역이다.
3. 오늘 출근시간이 없으면 빈 값을 그대로 노출하지 않고 placeholder 또는 숨김으로 처리한다.
4. 날짜 판별은 사용자의 로컬 캘린더 기준 `isDateInToday` 성격의 규칙으로 처리한다.
5. 다중 출근/퇴근 세션은 이번 범위에서 다루지 않고, 오늘 기록 1개만 존재한다고 가정한다.
