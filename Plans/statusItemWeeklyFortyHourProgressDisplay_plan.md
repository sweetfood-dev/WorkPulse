# WorkPulse 상태 아이템 주 40시간 기준 진행/초과 표시 계획 (`statusItemWeeklyFortyHourProgressDisplay_plan.md`)

## 요약
- 기준 대상은 `statusItemWeeklyWorkedTimeDisplay_plan.md`를 전제로 한 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 사용자는 이번 주 누적 근무시간이 주 40시간 기준으로 얼마나 부족한지, 혹은 얼마나 초과했는지를 즉시 확인할 수 있어야 한다.
- 부족 시간과 초과 시간은 숫자뿐 아니라 시각적으로도 명확히 구분되어야 한다.
- 첫 단계의 시각 강조는 상태 아이템 클릭 후 열리는 팝오버 요약 영역에서 제공한다.
- 이번 범위에서는 법정근로시간 정책 확장, 사용자별 목표시간 설정, 휴게시간 규칙은 제외한다.

### 목표 (Goal)
사용자가 WorkPulse를 열면 이번 주 기준으로 `남은 시간` 또는 `초과 시간`을 바로 이해할 수 있고, 색상 또는 상태 배지 차이만 봐도 현재 주간 근무 상태를 직관적으로 파악할 수 있도록 한다.

### 제약 사항 (Constraints)
- 핵심 계산 로직은 Foundation 기반 순수 로직으로 테스트 가능해야 한다.
- AppKit 의존은 팝오버/상태 요약 표시 계층에 한정한다.
- 주간 기준 목표시간은 40시간으로 고정한다.
- 첫 단계에서는 개인별 목표시간 커스터마이징을 허용하지 않는다.
- 메뉴 막대 본문은 공간 제약이 크므로 색상 기반 강조의 주 표시 위치는 팝오버 요약 영역으로 한정한다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 주 40시간 기준 진행 상태를 표현하는 순수 타입을 추가한다.
- 예: `WeeklyTargetProgress`
- 속성 예시: `workedDuration`, `targetDuration`, `remainingDuration`, `overtimeDuration`, `status`

2. 표시 상태를 위한 semantic token 또는 enum을 추가한다.
- 예: `WeeklyTargetProgressStatus`
- case 예시: `underTarget`, `metTarget`, `overTarget`

3. 팝오버 요약 표시 모델을 추가한다.
- 예: `WeeklyTargetProgressDisplayModel`
- 속성 예시: `titleText`, `valueText`, `accentStyle`

4. AppKit 표시 계층에서 색상 결정을 직접 숫자 비교로 하지 않도록 semantic token 기반으로 연결한다.
- 예: `accentStyle` → `NSColor.systemOrange` / `NSColor.systemGreen` / `NSColor.systemRed`

## 테스트 케이스 및 시나리오
1. 40시간 미만
- Given: 이번 주 누적 근무시간이 31시간 30분
- When: 목표 대비 표시 계산
- Then: `채워야 하는 시간 08:30` 표시, 부족 상태 style 적용

2. 40시간 정확히 충족
- Given: 이번 주 누적 근무시간이 40시간
- When: 목표 대비 표시 계산
- Then: 완료 상태 표시, 중립 또는 완료 style 적용

3. 40시간 초과
- Given: 이번 주 누적 근무시간이 44시간 15분
- When: 목표 대비 표시 계산
- Then: `초과된 시간 04:15` 표시, 초과 상태 style 적용

4. 주간 값 갱신
- Given: 이번 주 근무시간이 갱신됨
- When: 목표 대비 표시를 refresh
- Then: 숫자와 style이 최신 누적값 기준으로 갱신

5. 기존 주간 총합과 공존
- Given: 팝오버에 이번 주 총 근무시간 요약이 이미 존재
- When: 목표 대비 표시가 추가됨
- Then: 총합 표시와 목표 대비 표시가 서로 다른 의미로 함께 노출

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “주 40시간”은 정확히 40시간 0분으로 해석한다.
2. `채워야 하는 시간`은 `40시간 - 이번 주 누적 근무시간`으로 계산한다.
3. `초과된 시간`은 `이번 주 누적 근무시간 - 40시간`으로 계산한다.
4. 40시간 정확히 달성한 경우는 부족/초과가 아닌 별도 완료 상태로 표시한다.
5. 색상 강조는 첫 단계에서 팝오버 요약 영역에 우선 적용하고, 메뉴 막대 본문은 공간/시스템 제약상 보조 텍스트만 사용한다.
6. 기본 색상 의미는 부족=주의 계열, 완료=성공 계열, 초과=강조 계열로 두되 실제 `NSColor` 매핑은 구현 단계에서 플랫폼 가독성을 우선해 결정한다.
