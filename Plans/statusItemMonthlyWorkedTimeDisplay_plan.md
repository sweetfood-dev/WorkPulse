# WorkPulse 상태 아이템 월간 근무시간 집계 표시 계획 (`statusItemMonthlyWorkedTimeDisplay_plan.md`)

## 요약
- 기준 대상은 `statusItemAttendanceTimeInput_plan.md`, `statusItemWorkedTimeDisplay_plan.md`, `statusItemWeeklyWorkedTimeDisplay_plan.md`를 전제로 한 macOS 전용 `WorkPulse` AppKit 앱으로 고정한다.
- 사용자는 이번 달 누적 근무시간을 메뉴 막대 상태 아이템 클릭 후 열리는 팝오버에서 확인할 수 있어야 한다.
- 월간 집계는 로컬에 저장된 일별 출근/퇴근 기록을 현재 달 범위로 합산해 계산한다.
- 첫 단계에서는 월간 총 근무시간만 제공하고, 목표시간 비교나 월별 차트는 제외한다.
- 이번 범위에서는 서버 동기화, 다중 사용자, 월간 리포트 내보내기는 다루지 않는다.

### 목표 (Goal)
사용자가 WorkPulse를 열면 이번 달 누적 근무시간을 즉시 확인할 수 있고, 일별 기록이 저장되거나 수정될 때 월간 집계가 바로 갱신되도록 한다.

### 제약 사항 (Constraints)
- 월간 집계 계산 로직은 Foundation 기반 순수 로직으로 테스트 가능해야 한다.
- AppKit 의존은 상태 아이템/팝오버 표시 계층에 한정한다.
- 집계 대상은 로컬 저장소에 보관된 일별 출근/퇴근 완료 기록으로 제한한다.
- 현재 달 판단은 사용자의 현재 로컬 캘린더와 타임존 기준으로 처리한다.
- 첫 단계에서는 월간 총합만 표시하고, 일별 breakdown UI는 추가하지 않는다.

## 공개 인터페이스/타입 변경(계획 기준)
1. 월간 집계 계산용 순수 타입을 추가한다.
- 예: `MonthlyWorkedTimeCalculator`

2. 월간 표시용 Render Model을 추가한다.
- 예: `MonthlyWorkedTimeDisplayModel`
- 속성 예시: `text`, `isPlaceholder`

3. 저장된 기록에서 현재 달 범위를 조회하는 저장소 메서드를 추가한다.
- 예: `records(inMonthContaining:)`

4. 팝오버 요약 영역에 월간 근무시간 표시 섹션을 추가한다.
- 예: "이번 달: 128:30"

## 행동 변경 (Behavioral Changes - TDD)
*지시가 있을 때마다 아래의 체크되지 않은 첫 번째 테스트를 찾아 Red 테스트를 작성하고, 이를 통과시키는 최소 코드만 작성한다.*

- [x] 이번 달에 저장된 근무 기록이 없으면 월간 근무시간은 placeholder로 표시된다.
- [x] 이번 달에 하루 근무 기록 1개가 있으면 해당 시간만큼 월간 합계가 표시된다.
- [x] 이번 달에 여러 날의 근무 기록이 있으면 각 일자의 근무시간 합계가 표시된다.
- [x] 현재 달 범위를 벗어난 지난달/다음달 기록은 월간 합계에서 제외된다.
- [x] 동일 날짜의 기록이 수정되면 이전 값이 중복 합산되지 않고 최신 값 기준으로 다시 계산된다.
- [x] 출근/퇴근시간 저장 또는 수정 후 월간 근무시간 표시가 즉시 갱신된다.
- [x] 월간 집계 추가가 기존 현재/주간 근무시간 표시를 깨지 않는다.
- [x] 월간 집계 추가가 기존 macOS 앱 빌드 성공을 깨지 않는다.

- [x] 저장소의 기간 범위 조회를 주간 전용 구현에서 재사용 가능한 형태로 일반화한다.

## 테스트 케이스 및 시나리오
1. 기록 없음
- Given: 이번 달에 저장된 근무 기록이 없음
- When: 월간 합계 계산
- Then: placeholder 문자열 표시

2. 단일 일자 집계
- Given: 이번 달 특정 날짜 근무 기록 1개가 저장됨
- When: 월간 합계 계산
- Then: 해당 일자의 근무시간이 그대로 표시

3. 복수 일자 합계
- Given: 이번 달 여러 날짜의 근무 기록이 저장됨
- When: 월간 합계 계산
- Then: 각 날짜 근무시간의 합이 표시

4. 월 경계 제외
- Given: 지난달 기록과 이번 달 기록이 함께 저장됨
- When: 월간 합계 계산
- Then: 이번 달 기록만 집계

5. 기록 수정 반영
- Given: 이번 달 특정 날짜 기록이 이미 저장되어 있음
- When: 사용자가 그 날짜 출근/퇴근시간을 수정
- Then: 월간 합계가 최신 기록 기준으로 즉시 갱신

6. 빌드 검증
- Given: macOS 타깃 빌드 수행
- When: `xcodebuild -destination 'platform=macOS' build`
- Then: 빌드 성공

## 명시적 가정 및 기본값
1. “월간 집계”는 사용자의 현재 로컬 캘린더 기준 현재 달의 누적 근무시간으로 해석한다.
2. 월간 합계는 기본적으로 일별 `퇴근시간 - 출근시간`의 합으로 계산한다.
3. 같은 날짜 기록은 1개만 유지하고, 수정 시 덮어쓴다.
4. 진행 중 근무 포함 여부는 별도 요구가 없으면 확정된 출근/퇴근 기록만 집계하는 보수적 기본값을 사용한다.
5. 월간 총합은 첫 단계에서 팝오버 요약 영역에 표시하고, 메뉴 막대 본문에는 직접 노출하지 않는다.
6. 휴게시간 차감, 자정 넘김, 다중 세션 합산, 월간 목표시간 비교는 이번 범위에서 다루지 않는다.
