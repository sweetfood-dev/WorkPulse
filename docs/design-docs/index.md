# design-docs/index.md

## Purpose

이 디렉터리는 장기적으로 남겨야 하는 디자인 판단 근거를 저장합니다.

## What Belongs Here

- 대안 비교
- 정보 구조 결정
- 상호작용 모델 선택 이유
- 반복될 수 있는 디자인 판단의 근거

## What Does Not Belong Here

- 단순 작업 체크리스트
- 구현 세부사항 중심 메모
- 제품 요구사항 문서 자체

## Recommended Shape

각 문서는 가능하면 아래를 포함합니다.

- 문제와 맥락
- 목표와 비목표
- 대안 비교
- 선택한 방향
- 후속 리스크
