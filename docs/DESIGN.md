# DESIGN.md

## Purpose

이 문서는 제품과 상호작용 디자인 판단의 공통 기준을 정의합니다.

## Principles

- 사용자는 시스템 상태를 읽을 수 있어야 한다
- 기본 경로는 단순해야 한다
- 되돌리기 어려운 행동은 설계가 먼저 보호해야 한다
- 중요한 것과 눈에 띄는 것을 혼동하지 않는다
- 시스템 전체의 일관성이 개별 화면 취향보다 중요하다

## When To Create A Design Doc

- 합리적인 대안이 둘 이상 있을 때
- 정보 구조를 새로 정의할 때
- 상호작용 모델이 기존 패턴을 깨거나 확장할 때
- 이후 여러 기능에 반복될 판단 기준이 생길 때

## Review Questions

- 사용자가 현재 상태와 다음 가능한 행동을 읽을 수 있는가
- 기본 경로가 예외 경로보다 더 단순한가
- 되돌리기 어려운 행동을 설계가 먼저 보호하는가
- 새로운 패턴이 기존 시스템 일관성을 깨면 그 이유가 충분한가

위 질문이 애매하면 디자인 판단을 구현 안으로 숨기지 말고 `docs/design-docs/`로 승격합니다.

## Where Those Docs Live

개별 디자인 근거 문서는 `docs/design-docs/`에 둡니다.
